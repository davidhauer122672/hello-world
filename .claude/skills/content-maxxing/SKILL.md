# Content Maxxing Skill

Extract maximum surface area, signal density, and revenue conversion from a single pillar asset. One moment becomes an atomic content fleet — engineered, tested, distributed, and measured.

## Core Doctrine

1. **Capture once. Deploy infinitely.** Every pillar asset is a mine, not a post.
2. **Atomize to the smallest valuable unit.** A 60-min recording yields ~40 publishable atoms.
3. **Distribute on native grammar.** No cross-posting. Rebuild each atom for the host platform.
4. **Measure to the conversion, not the like.** Pipeline movement is the only surviving KPI.
5. **Compress relentlessly.** One sentence beats a paragraph. Quality density wins.

## When to Invoke

- User provides a pillar asset (video transcript, essay, interview, event recording, deal story)
- User provides a topic + target segment (Absentee Owner / Luxury Homeowner / Portfolio Investor / STR Owner / Seller / Relocating Buyer)
- User provides a revenue objective (lead-gen / authority / reactivation / listing acquisition)

If any input is missing or ambiguous, request clarification **once** with a numbered list. Do not assume.

## Execution Protocol (7 Phases — run in sequence, no pauses)

**Phase 1 — Research (IQVIA).** Target segment, pain point, objection, decision trigger. Surface 3 competitive angles; identify the gap Coastal Key will own. Output: Audience Brief (≤120 words) + Competitive Gap (1 sentence) + Decision Trigger (1 sentence).

**Phase 2 — Architecture (Siemens + SpaceX).** Decompose pillar into atomic value units. Map each to highest-leverage format and platform. Output: Atomic Inventory Table [Atom # | Value Unit | Format | Platform | Segment | KPI].

**Phase 3 — Design (Chanel / LVMH + David Yurman).** Per atom: composition, palette (Navy `#1B2A4A`, Sand `#D2C4A0`, Ocean Blue `#2E86AB`), typography, signature element. Restraint test: if anything can be removed without information loss, remove it. Output: Visual Direction per atom (≤25 words each).

**Phase 4 — Production (Walmart + Amazon).** Final copy per atom. Hook in 1.5 seconds. Payoff before the scroll. One CTA. Compress every line. Output: Copy-paste-ready block per atom.

**Phase 5 — Automation (UiPath).** Generate publish payload: schedule, captions, hashtags, alt-text, UTM parameters. Wire into Buffer → Airtable (base `appUSnNgpDkcEOzhN`) → Slack `#marketing-ops` → `ck-api-gateway /v1/social/*`. Output: Publish Manifest (JSON-ready) + Trigger Sequence.

**Phase 6 — Testing (DeviQA).** Run each atom through gates: voice match · factual integrity · governance compliance · conversion intent (≤2 actions to KPI) · platform native (length/ratio/hashtag). Output: QA Report per atom [Pass / Revise / Reject + Reason].

**Phase 7 — Polish (Sovereign).** Cut every word that does not earn its space. If it sounds like marketing, rewrite until it sounds like authority. Confirm CEO Freedom Protocol (fleet publishes without further intervention). Output: Fleet Summary [Atoms Approved | Reach Estimate | Pipeline KPI Target | Publish Window].

## Anti-Patterns (auto-reject)

- Cross-posting identical copy across platforms
- Hashtag stuffing beyond platform native count
- Hooks longer than 1.5 seconds read/watch time
- "Check out", "Excited to announce", "We are thrilled", any exclamation point
- Vanity metrics as KPI (likes, impressions without conversion path)
- Third-party vendor name-drops in public copy
- More than one CTA per atom
- Any atom that cannot answer: "What does the prospect do in the next 30 seconds?"

## Deliverable Format (single response)

1. Audience Brief
2. Competitive Gap + Decision Trigger
3. Atomic Inventory Table
4. Visual Direction
5. Final Copy (per atom, paste-ready)
6. Publish Manifest (JSON)
7. QA Report
8. Fleet Summary + Execution Command
