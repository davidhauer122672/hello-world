# Claude Secrets Skill

Extract sovereign-grade output from Claude by controlling the four levers most users never touch: **Context Architecture, Instruction Density, Output Contract, Iteration Protocol**. Treat Claude as an executive staff of specialized experts — briefed properly, constrained precisely, measured against deliverable standards.

## When to Invoke

- User describes a task and wants an elite prompt (not an answer)
- User needs to convert a vague ask into a production prompt
- User wants to version a winning prompt into the Coastal Key library
- User has a high-stakes Claude task where first-pass quality matters

## The Ten Operator Secrets

1. **Role before request** — name the expert Claude is being
2. **Context beats cleverness** — paste facts, don't reference them
3. **Constraints produce quality** — word caps, format locks, banned phrases raise output 3–10x
4. **Extended thinking for high stakes** — invoke reasoning when cost-of-wrong is meaningful
5. **One deliverable per prompt** — atomize compound requests
6. **Show the target** — include positive and anti-exemplars
7. **Anchor the voice** — paste style samples, don't describe them
8. **Chain, don't bundle** — sequence prompts for multi-stage work
9. **Review with a second role** — self-critique against a written rubric kills 80% of defects
10. **Save every winning prompt** — versioned prompts are capital, not notes

Fortune 500 · Red Bull Racing · SpaceX · Ferrari · IQVIA · UiPath · DeviQA · AECON · Siemens · Alphabet/Deloitte/PwC/EY/KPMG (audit) · Alphabet (risk) · Cooley/Gibson Dunn/Latham & Watkins (legal tempering) · Coastal Key Sovereign Governance voice.

## The Four Levers (apply to every prompt)

**Lever 1 — Context Architecture.** Paste facts, not references. Attach relevant CLAUDE.md sections, target files, prior artifacts. State what has been tried and ruled out.

**Lever 2 — Instruction Density.** Role + Context + Task + Constraints + Output Format + Quality Gates + Trigger. One deliverable per prompt. Word caps and format locks on every output.

**Lever 3 — Output Contract.** Exact sections, tables, word counts, tone, banned phrases. Include exemplars where possible. Forbid soft hedges, question restatements, and "happy to help" preambles.

**Lever 4 — Iteration Protocol.** High-stakes: generate → critique → revise. Production: chain prompts. Creative: produce 3 candidates, score, select or merge.

## Execution Protocol (6 Phases)

**Phase 1 — Brief (IQVIA).** Identify deliverable, audience, stakes, dependencies. Output: Brief Statement (≤80 words).

**Phase 2 — Architecture (SpaceX + Siemens).** Apply Four Levers. Choose single-prompt or chained-prompt pattern. If chained, map the critical path. Output: Prompt Architecture.

**Phase 3 — Draft (Ferrari).** Write the prompt. One deliverable. Explicit role, context, task, constraints, output contract, trigger. Output: Prompt v1 in code block.

**Phase 4 — Audit (Deloitte / PwC / EY / KPMG / Alphabet).** Inspect against control framework: role assigned · context pasted not referenced · deliverable atomized · constraints explicit · output contract specified · risk statement included · legal tempering applied where required · Sovereign Governance tone. Output: Audit Report [Pass / Revise per item].

**Phase 5 — Legal Temper (Cooley / Gibson Dunn / Latham & Watkins).** If output touches contracts, disclosures, investment claims, client-facing copy, or regulated subject: apply hedged language, jurisdictional notes, disclaimers. If pure internal ops: skip. Output: Tempered prompt or "N/A — internal only".

**Phase 6 — Release & Version (UiPath).** Name the prompt. Assign version. Declare owner, reuse pattern, expected KPI. File into prompt library. Output: Release Manifest [Name | Version | Owner | Reuse Pattern | KPI | Location].

## Deliverable Format (single response)

1. Brief Statement
2. Prompt Architecture
3. Final Prompt (code block, paste-ready)
4. Audit Report
5. Legal Temper Note
6. Release Manifest

## Anti-Patterns (auto-reject)

- "Help me with…" openings → replace with direct role
- Vague adjectives ("good", "nice", "better") → replace with measurable criteria
- Open-ended output ("write a summary") → replace with bounded structure
- Compound requests → atomize
- Missing context references → paste the context
- Style described, not shown → paste a style sample
- Unversioned prompt → a prompt that works once and isn't versioned never existed

## Standard Legal Temper Pattern

> "Projections are based on [dataset] and are not guarantees of future performance. Past results do not ensure future outcomes. Not legal, financial, or tax advice. Specific to [jurisdiction]. Consult licensed counsel."
